# Codebase Mapping Prompt

Analyze the complete codebase.

Produce:
1. Project type
2. Real technical stack
3. Folder structure
4. Entry points
5. Main modules
6. Business services
7. Data models
8. Configuration
9. Secrets or sensitive values
10. Existing commands
11. Existing tests
12. Naming conventions
13. Real architecture
14. Main flows
15. Stable areas
16. Fragile areas
17. Probable dead code
18. Visible but disconnected features
19. Hardcoded behaviors
20. AI-related components
21. Model/provider-related components
22. First safe intervention point

Constraints:
- Cite inspected files.
- Separate facts and assumptions.
- Do not invent files.
- Do not summarize the README as truth.
