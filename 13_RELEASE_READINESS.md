# Release Readiness

Go criteria:
1. Build passes.
2. Critical tests pass.
3. No secrets.
4. No blocking hardcoding.
5. README reflects code reality.
6. Installation tested.
7. Minimal demo works.
8. P0 bugs closed.
9. Logs are clean.
10. Changelog written.
11. Version tagged.
12. Rollback possible.
13. AI model configuration documented.
14. Cost limits configured.
15. Rate limits configured.
16. Fallback behavior documented.
17. Privacy risks documented.

No-go criteria:
1. Core feature does not work.
2. Launch fails.
3. Mocked data presented as real.
4. External dependency undocumented.
5. Excessive manual configuration.
6. Tests impossible without clear reason.
7. Marketing claim not proven.
8. Paid API can spend without limits.
9. Sensitive data sent to remote models without policy.
